# Project Polish Skill — 项目整理四件套

当用户请求对某个 `-projects/` 下的项目文件夹做零碎收尾时（如"整理这个项目"、"整理这个文件夹"、"polish"、"收尾"），执行以下四步。设计意图：项目的代码写完了，但文件命名随意、缺少 README 和 TODO——一键标准化。

## 设计原则

- **只做机械整理，不改代码逻辑。** 加 docstring 例外（经验化时）。
- **四个步骤按需执行。** 如果某步骤无可操作内容，跳过并说明原因。
- **一次性展示计划，确认后批量执行。** 除非用户明确说"直接做"。

---

## 步骤 1：文件重命名检查

### 1.1 列出目标文件夹下所有 `.py` 文件

### 1.2 自动判定模式（复用 rename-folder 逻辑）

- **模式 A（占位符命名）**：文件名匹配 `demo*`、`^\d+\.py$`、`test*`、`练习*`、`new*`、`temp*` 之一
- **模式 B（校准统一）**：文件名已有语义但不够准确或风格不统一
- **模式 C（已规范）**：文件名已符合 `{难度}-{功能描述}.py` 格式 → 跳过

### 1.3 生成重命名计划

- 统一命名格式：`{难度}-{功能描述}.py`
- 难度层：`基础` / `进阶` / `实战`
- 功能描述：中文，≤10 字，不重复父文件夹名中的主题词

### 1.4 执行重命名（用户确认后）

- 优先 `git mv`，文件未被 git 追踪则退化 `mv`
- 路径含 `-` 开头时加 `./` 前缀

---

## 步骤 2：经验化脚本识别

### 2.1 检查可提取性

对每个 `.py` 文件判断：
- 是否是自包含的工具脚本（非项目主入口）？
- 其功能是否已有类似片段在 `_recipes/` 中？

### 2.2 判定结论

三种结论：
- **可提取** — 独立的工具脚本，功能通用，建议迁移到 `_recipes/`
- **不提取（已有）** — 功能已被 `_recipes/` 中某文件覆盖
- **不提取（耦合）** — 逻辑与项目特定细节（XPath、URL、业务）强耦合，独立后无复用价值

### 2.3 执行经验化（用户确认后）

对确认为可提取的文件，按 snippet-extract 流程执行：
1. 写入 `原项目` + `功能` docstring
2. 生成中文"我做了什么"文件名
3. 移动到 `_recipes/`

---

## 步骤 3：补充 README.md

### 3.1 检查是否存在

若已存在较完整的 README.md，仅补充缺失项（如完成日期）。若不存在，创建。

### 3.2 README 模板

```markdown
# {项目名} — {一句话功能定位}

**Date:** {完成日期}
**Author:** zigyr

## 概述
{2-3 句话说明项目干什么、技术路线}

## 技术链路 / 技术栈
{关键流程或技术表格}

## 目录结构
| 文件 | 说明 |
|------|------|
| ... | ... |

## 依赖
```bash
pip install ...
```

## 使用方式
{如何运行、输入什么、输出什么}

## 相关经验片段
{如果有已提取的 _recipes/ 片段，列出}
```

### 3.3 完成日期

取文件 docstring 中的 Date，或文件最早修改日期（git log），或今天。

---

## 步骤 4：补充 TODO.md

### 4.1 检查是否存在

若已有 TODO.md：保留已有条目，在已有框架基础上扩充。
若不存在：从零创建。

### 4.2 优化方向分类

按以下维度扫描项目，生成 TODO：

| 类别 | 典型条目 |
|------|----------|
| **功能完善** | 未覆盖的内容类型、自动化程度提升 |
| **下载稳定性** | 重试、超时、断点续传、进度条 |
| **内容质量** | 清晰度选择、格式筛选、去水印 |
| **工程化** | 配置文件、日志、异步并发、cookie 管理 |

### 4.3 格式

```markdown
# TODO — 日后优化方向

## {类别}

- [ ] **{条目}** — {简短说明}
```

---

## 执行流程总结

```
用户: "整理这个项目"
  │
  ├─ 步骤1: 扫描 .py → 判定模式 → 生成重命名计划
  ├─ 步骤2: 判断可经验化脚本 → 给出结论
  ├─ 步骤3: README.md 是否存在？→ 创建/补充
  └─ 步骤4: TODO.md 是否存在？→ 创建/扩充
  │
  └─ 展示完整计划，用户确认 → 批量执行
```

---

## 注意事项

- 仅处理 `-projects/` 下的项目文件夹。
- 不改代码逻辑，仅做文件级整理。
- 路径分隔符统一用 `/`。
- 目录名以 `-` 开头时，shell 命令加 `./` 前缀。
- 文件未被 git 追踪时，`git mv` 退化为 `mv`。
