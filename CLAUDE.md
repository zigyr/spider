# Spider — Python 爬虫学习与实战

基于 Python 的网络爬虫学习项目，涵盖从基础请求到高级逆向工程的全链路实践。

## 项目结构

| 目录 | 用途 |
|------|------|
| `-course/` | 课程笔记：多来源爬虫教程实战 |
| `-projects/` | 实战项目：图片 / 文字 / 视频爬取 |
| `_recipes/` | 个人经验片段：从项目中提炼的可复用脚本 |
| `_notes/` | 零散笔记与踩坑记录 |
| `journal/` | 自动生成的学习日志（勿手动编辑） |

## ⚙️ 自动日志记录

当用户在对话中请求任何 git 操作时（"帮我 git"、"git add"、"git commit"、"提交"、"push"等），在 git 操作完成后，**自动且静默**地分析变更并写入学习日志。流程如下：

1. 获取变更文件列表（优先 `git diff --cached --name-only`，其次 `git diff --name-only`）
2. 阅读关键新增/修改的 `.py` 文件，理解其内容和用途
3. 按文件路径自动归类到对应模块
4. 生成简洁学习摘要，追加到 `journal/YYYY-MM-DD.md`
5. **无需向用户确认**，静默完成

具体格式和执行规范见 `.claude/skills/journal.md`。

## ⚙️ 经验片段提取

当用户请求"经验化"某个脚本时（如"经验化这个文件"、"提取为经验"），自动执行以下流程：

1. 确认目标文件路径和功能描述
2. 在文件顶部写入来源标注 docstring（`原项目` + `功能`）
3. 生成中文"我做了什么"风格的文件名
4. 移动到 `_recipes/` 个人经验片段库
5. 用 `git mv` 保留 git 追溯

**设计意图**：从各项目的脚本中提炼通用经验，文件名即索引，扫一眼就知道这段代码能做什么。

具体格式和执行规范见 `.claude/skills/snippet-extract.md`。

## ⚙️ 文件夹重命名整理

当用户请求对某个文件夹下的脚本进行重命名整理时（如"整理这个文件夹"、"统一命名"），自动判定并执行：

- **模式 A — 从零命名**：文件名是占位符（`demo*.py`、`1.py` 等），完全基于代码内容生成中文名
- **模式 B — 校准统一**：文件名已有语义但不够准确或风格不统一，基于代码重新校准

统一命名框架：`{难度}-{功能描述}.py`（难度分 基础 / 进阶 / 实战 三级），父文件夹名已提供主题上下文，文件名不重复主题词。

**设计意图**：解决听课时临时命名（demo1、demo2…）导致回顾困难的问题，所有课程代码按统一标准可检索。

具体格式和执行规范见 `.claude/skills/rename-folder.md`。
